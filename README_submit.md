# 온라인테스트1 — 멀티모달 기반 근권부 환경 예측

온실 환경 시계열과 딸기 캐노피 다분광 영상으로부터
5분 단위 근권부 환경(배지 함수율·EC·온도)을 추정합니다.

> **설계 원칙**
> 데이터에 제어·관리 방식이 서로 다른 4개의 재배 그룹이 있음을 분석으로 확인했습니다.
> 각 날짜를 그 그룹 중 하나로 분류해 예측에 활용하며, 날짜 간 순서는 사용하지 않습니다.
> 각 날(DAT)을 독립 단위로 다루고, test 를 train 뒤에 잇지 않으며,
> test 의 어떤 값도 학습 특징에 넣지 않습니다.
> 오직 **train 에 나타나는 4개 그룹 패턴**만 학습해 test 에 적용합니다.

---

## 1. 실행 방법

```bash
docker build -t submission .
docker run --rm \
  -v "$PWD/input:/app/input" \
  -v "$PWD/output:/app/output" \
  submission sh -c "python train.py && python inference.py"
```

- `train.py` → `model/` 에 가중치 및 구조 아티팩트 생성
- `inference.py` → `output/submission.csv` 생성 (형식은 `train_y.csv` 와 동일, 3,456행)
- 상대 경로 `input/`, `output/` 를 고정 사용하며 수정하지 않았습니다.
- 난수 시드(42), LightGBM `deterministic=True`, `OMP_NUM_THREADS=4` 고정으로
  동일 입력에 대해 항상 동일한 `submission.csv` 가 재생성됩니다.

**소요 시간** (4코어 기준): `train.py` 약 3분 (다분광 639세션 처리 포함), `inference.py` 약 1분.

## 2. 데이터 사용 범위 — 규칙(5.1.5) 준수 사항

| 항목 | 준수 내용 |
|---|---|
| 외부 데이터 | **일절 사용하지 않았습니다.** 학습·추론 모두 `input/dataset/` 안의 제공 데이터만 사용합니다. |
| 사전학습 가중치 | **사용하지 않았습니다.** 모든 가중치는 `train.py` 가 제공 데이터로부터 생성합니다. |
| 정답 참조 | 평가 정답(`test_y`)은 **어떤 형태로도 참조하지 않습니다.** |
| 사람 개입 | 없습니다. 추론은 전적으로 코드로만 이루어집니다. |
| 하드코딩 | **상수를 하드코딩하지 않았습니다.** 그룹 구조는 코드가 데이터로부터 도출합니다. |
| 날짜 간 순서 | **사용하지 않습니다.** 각 날을 독립 단위로 다루며 test 를 train 뒤에 잇지 않습니다. |
| 학습–평가 분리 | **학습 특징은 train 환경만으로 계산됩니다.** test 의 값은 학습에 흘러들지 않습니다. |
| 파일/폴더 구조 | 5.1.4 규격을 준수했습니다. `input/`, `output/` 은 빈 폴더로 제출합니다. |

평가 구간의 `test_X`(환경 시계열)는 오직 **"각 test 날짜가 어느 그룹인지" 분류**에만 쓰입니다
(비지도, 정답 미사용, 제공 데이터 범위). 이는 학습된 KMeans 로 test 날짜를 분류하는
표준 절차이며, test 데이터가 train 모델의 가중치나 특징에 관여하지 않습니다.

---

## 3. 핵심 관찰 — 4개 재배 그룹 구조

`train_X` 의 외부 기상(온도·습도·일사·풍향·풍속·강우)을 DAT별로 비교하면, 6일치 기상이
서로 다른 4개 DAT 구간에서 소수점까지 동일하게 반복됩니다. 반면 내부 기후와 제어 이력은
구간마다 뚜렷이 다릅니다(한 구간은 천창을 전혀 열지 않고, 다른 구간은 76% 가동).

→ 데이터에 **제어·관리 방식이 서로 다른 4개의 재배 그룹**이 존재하며, 근권부 EC 가
그룹마다 뚜렷이 다른 수준을 가집니다. 각 날짜는 이 4개 그룹 중 하나에 속합니다.

### 코드가 도출하는 것 (`src/structure.py`)

1. **외부기상 그룹** — 외부 기상이 사실상 동일한 날들을 union-find 로 묶습니다.
   이 그룹의 최대 크기가 곧 서로 구분되는 재배 그룹 수(=4)입니다.
2. **재배 그룹(zone)** — 날짜별 제어·내부기후 지문(제어 이력 가동률 + 내부 기후)을 KMeans 로
   군집화하고, 외부기상이 같은 날들은 서로 다른 그룹이 되도록 Hungarian 배정합니다.

test 날짜들도 이 학습된 군집으로 분류되어, train 의 어느 그룹에 해당하는지 정해집니다.
도출 결과는 `model/structure.csv` 에 저장됩니다.

## 4. 특징 설계 — 그룹 패턴 + 당일 정보만

- **그룹 원핫 + 그룹별 운영 요약**: 그날 그 그룹의 제어 이력 가동률과 내부 기후 수준.
- **그룹 × 하루 중 시각 기후값 (target encoding)**: 근권부 상태의 일중 곡선이 그룹마다
  다릅니다. 그 곡선을 학습 라벨에서 추정해 특징으로 줍니다.
  교차검증에서는 **각 fold 의 학습 부분만으로 계산**하여 누수를 차단했습니다.
- **당일(자정 리셋) 이력**: 30분~6시간 이동평균, 편차, 당일 누적 일사·VPD.
  **모든 이력은 하루 경계에서 리셋**되어 자정을 넘지 않습니다 — 서로 다른 그룹·서로 다른
  날의 데이터가 섞이지 않습니다.
- **시각(sin/cos)** 과 물리 파생량(VPD, 내외부 온도차 등).

DAT 번호 자체는 연속 특징으로 **사용하지 않습니다**(여러 그룹이 섞여 있는 축이므로).

### 다분광 (`src/msfeat.py`)

제출은 5분 격자 **전체**(야간 포함)를 채워야 하는데 영상은 낮 12~17시의 **3.9%** 시점에만
있습니다. 따라서 분광을 **하루·그룹 단위 캐노피 요약**으로 만들어 그날의 모든 시각에
브로드캐스트합니다. 전처리는 **암전류 보정 + DN_D1**(원시 DN 의 파장축 1차 미분)을 씁니다.
교차검증 결과 분광은 **함수율에만** 도움되어 함수율에만 사용하고, EC·배지온도에서는 제외합니다.

## 5. 모델과 검증

**모델:** 타깃별 LightGBM 회귀 3개.

**검증:** 원본 **DAT 단위 GroupKFold (6-fold)**. 하루(DAT)를 통째로 빼는 표준 그룹 CV 로,
같은 날의 인접 시점이 학습·검증에 갈리는 누수를 막습니다.

| 타깃 | 특징 집합 | DAT-6fold RMSE |
|---|---|---|
| `soil_moisture` | 전체 (**분광 포함**) | **0.734** |
| `soil_ec` | 전체 − 분광 | **0.052** |
| `soil_temp` | 전체 − 분광 | **0.669** |

**출력 안전장치:** 예측값은 학습 관측 범위(±10% 여유)로 클립하며, NaN/inf 가 없도록
최종 검사 후 저장합니다.

## 6. 파일 구성

```
/app/
├─ Dockerfile           실행 환경 (베이스 이미지·의존성 버전 고정)
├─ train.py             학습 진입점
├─ inference.py         답안 생성 진입점 → output/submission.csv
├─ README.md            본 문서
├─ .env                 환경변수 (외부 API 미사용)
├─ src/
│  ├─ structure.py      외부기상 그룹 → 재배 그룹 군집
│  ├─ msfeat.py         다분광 큐브 특징 추출 (암전류 보정 + DN_D1)
│  └─ features.py       5분 격자 특징 (그룹 패턴 + 당일 이력)
├─ model/               학습 가중치 및 구조 아티팩트
├─ input/               (운영진 주입) 빈 폴더로 제출
└─ output/              inference.py 산출물. 빈 폴더로 제출
```
