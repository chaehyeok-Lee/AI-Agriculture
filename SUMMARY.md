**🚨 문제**: test 실제 예측 뭉침 — 고유값 104/3456개뿐, 최다값 39%(1341행) 차지
- 🆕 **실제 재현·시각화 완료** (`10_ec_clumping.py`, 로컬에서 train.py→inference.py 재실행해 검증): 최다값 0.3920이 1,341개(38.8%), 2위값 0.4235가 약 920개(27%) — **상위 2개 값이 전체의 65%**를 차지
- 🆕 예측 시계열을 보면 완전히 평평하진 않고 **하루 주기 네모 패턴이 12일 내내 반복**됨 → 모델이 `hour`(하루 안 패턴)는 어느 정도 반영하지만, train에서 봤던 날짜별 큰 폭 변화(0.35~2.9)는 전혀 못 담고 좁은 범위(0.39~0.435)에 갇힘 — "하루 잔물결은 흉내내지만 며칠 단위 큰 파도는 놓친다"

**원인 (누적 확인)**
1. day_num 외삽 (train 109~134 / test 135~146, 완전 비중첩)
2. **날씨(X) 데이터 자체가 복제됨** — train 9종 템플릿 / test 3종 템플릿(3일 주기 반복), train-test 겹치는 템플릿 0개
3. 피처 15개 중 10개가 test에서 train 범위 이탈 (계절 차이)

**시도했다 실패**
- 선형+LightGBM 하이브리드 → val 0.48로 악화 + 음수EC 발생 → 폐기
- LightGBM 보수화 → 더 악화(값 1개로 붕괴) → 폐기
- → **현재 원래 설정 유지 중**

**⚠️ 피처 함정**: "최근 EC 추세" 피처는 test에 EC 실측값이 아예 없어서 **계산 불가** (val만 좋아 보이는 함정) → X변수(날씨/구동기) 추세로 대체해야 함, **미구현**

**다음 시도 후보**: X변수 추세 피처 구현 / 화방 주기 가설 검증

## 🔧 3개 타깃 공통 — 구조적 발견

**날씨(X) 데이터 복제** (soil_ec에 가장 큰 영향, 나머지도 영향권)
- train 26일 중 실제 고유 날씨는 9종류뿐 (23일이 복제본)
- test 12일은 3종류가 정확히 3일 주기로 반복
- train·test 겹치는 템플릿 0개 — **파일 손상 아님**, 정답(y)은 매일 독립적으로 다름 확인됨
- 구동기/내부환경은 복제 안 됨 (circ_fan 등은 진짜 신호)

**train/test 범위 불일치 (3개 위험 컬럼, CLIP_RANGES 반영됨)**
| 컬럼 | train 범위 | test 범위 |
|---|---|---|
| greenhouse_roof_vent1 | 0~30 | 0~**59** |
| wind_speed_outside | 0~6.54 | 0~**8.28** |
| temperature(내부) | 1.5~26.1 | **-0.2**~24.6 |

**내부온도 쌍봉분포 원인 확정**: 낮 반응성은 동일(상관 0.746 vs 0.753), **야간만 1.46℃ 더 추움**(train 8.68 vs test 7.22)

**타깃 3종 상호 상관관계** (`11_target_corr.py`) — 전부 약함(|0.2| 이하), 서로 크게 얽혀있지 않음 → 지금처럼 타깃별 독립 모델 3개 쓰는 게 합리적
| | soil_moisture | soil_ec | soil_temp |
|---|---|---|---|
| soil_moisture | 1.00 | -0.13 | -0.12 |
| soil_ec | -0.13 | 1.00 | 0.20 |
| soil_temp | -0.12 | 0.20 | 1.00 |

**다분광(이미지)**: 현재 베이스라인 미반영 (마스킹/보정 안 된 날것이라 성능개선 없음, env-only와 env+ms 비교 시 유의미한 차이 없음). 위치0는 표본 작아 유의성 미검증. 실제로 쓰게 되면 **soil_ec에 우선 시도** — day_num/날씨처럼 "달력 기반" 정보가 아니라 식물 상태를 직접 관찰하는 정보라, 달력 정보가 안 통하는 soil_ec에 상대적으로 도움될 가능성이 가장 높음(단, 실제 검증 전까지는 가설)

**모델 선택**: LightGBM 유지 결정. 데이터가 26일(7,488행)뿐이라 딥러닝엔 부족, 선형모델은 계단식 비선형 패턴을 못 잡아서(하이브리드 시도 실패) 부적합, RandomForest/XGBoost/CatBoost는 LightGBM과 근본적으로 같은 한계(외삽 불가)라 교체 실익 없음. **문제는 모델 종류가 아니라 "학습 못 한 날짜로 외삽해야 하는 구조" 자체** → 모델 교체보다 피처(추세, 이미지) 보완이 맞는 방향
